# M5.5 — Design system foundation + primitives

Goal: land the "Sam Young" monospace-editorial design system (from the **The
Wire** handoff) as a reusable foundation — tokens, fonts, primitives, and a
`/style` proof page — so m5's feed UI can assemble it later. **No feed/data
wiring here** (m5's UI waits).

Spec: `../../design-docs/rankwire-design-system.md`. Read the bundled `fonts`
and `css` getting-started guides first (Tailwind v4 `@theme`, `next/font`).

Steps:

1. Fonts (`app/layout.tsx`): add `JetBrains_Mono`, `Libre_Caslon_Display` (400),
   `Libre_Caslon_Text` (400/700, italic) via `next/font/google`; keep `Geist`.
   Drop `Geist_Mono` (mono → JetBrains). Expose all as CSS variables on `<html>`.
2. Tokens (`app/globals.css`): brand constants + semantic role vars (light
   defaults, dark via `prefers-color-scheme` **and** `[data-theme]` override).
   Map roles + font families + radii + shadows into `@theme inline`. Remove the
   scaffold `--background/--foreground` and the Arial `body` rule; body defaults
   to `bg-page text-fg`, sans. Add the `tickPulse` keyframe + reduced-motion
   guard.
3. Primitives (`app/components/ui/`): `StickerButton`, `SegmentedControl`,
   `Badge`, `Rule`/`DoubleRule`, `UnreadTicker`, `Masthead`, `EndMark` per the
   design doc §4 contracts. Interactive ones `"use client"`; static ones stay
   server components. Barrel `index.ts`.
4. `/style` (`app/style/page.tsx`): token swatches + every primitive, rendered in
   both a light and a dark `data-theme` panel. Mock inline data only; no session,
   no DB. Public (not behind the feed's auth gate).
5. Keep the existing `Header` working (it uses `zinc-*` utilities — leave as-is;
   m5 restyles it). Ensure `bun run check` stays green.

Acceptance:

- `bun run dev`, visit `/style`: nameplate in Caslon display, mono labels in
  JetBrains, sticker buttons show the hard offset + press-flat physics, segmented
  control toggles, ticker bars pulse (and freeze under reduced-motion), both
  light and dark panels render correctly.
- No regression on existing pages (`/`, `/signin`) — they still build and render.
- `bun run check` green.

Out of scope: `FeedList`/`ArticleCard` and any DB/session reads (m5); vote wiring
(m6); digest/mini-masthead (m7); the email-digest section (dropped from product);
a user-facing theme toggle.
